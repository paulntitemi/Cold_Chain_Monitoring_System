# ColdTrack ESP32 - AWS IoT Core Certificates

This directory is reserved for AWS IoT Core TLS certificates used by the ESP32 firmware. The certificates themselves **must not** be committed to version control.

## Required Certificate Files

To authenticate your ESP32 device with AWS IoT Core (mutual TLS), you need three files:

| File | Description | Source |
|------|-------------|--------|
| `AmazonRootCA1.pem` | Amazon Root CA 1 (server trust) | [Download from Amazon Trust](https://www.amazontrust.com/repository/AmazonRootCA1.pem) |
| `device-certificate.pem.crt` | Device certificate | Generated by AWS IoT Core |
| `device-private.pem.key` | Device private key | Generated by AWS IoT Core |

## How to Obtain Certificates from AWS IoT Core

### Step 1: Create an IoT Thing

1. Open the [AWS IoT Core console](https://eu-west-1.console.aws.amazon.com/iot/home?region=eu-west-1) (eu-west-1 region).
2. Navigate to **Manage > All devices > Things**.
3. Click **Create things** > **Create single thing**.
4. Enter the Thing name (this must match `DEVICE_ID` in `config.h`, e.g. `ESP32_TEST_002`).
5. Click **Next**.

### Step 2: Generate Certificates

1. On the "Configure device certificate" page, select **Auto-generate a new certificate (recommended)**.
2. Click **Next**.

### Step 3: Attach a Policy

1. Create or select an IoT policy that allows your device to connect, publish, subscribe, and receive on the required topics. Example policy:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "iot:Connect",
      "Resource": "arn:aws:iot:eu-west-1:ACCOUNT_ID:client/${iot:Connection.Thing.ThingName}"
    },
    {
      "Effect": "Allow",
      "Action": "iot:Publish",
      "Resource": [
        "arn:aws:iot:eu-west-1:ACCOUNT_ID:topic/coldtrack/sensors/${iot:Connection.Thing.ThingName}/*"
      ]
    },
    {
      "Effect": "Allow",
      "Action": "iot:Subscribe",
      "Resource": [
        "arn:aws:iot:eu-west-1:ACCOUNT_ID:topicfilter/coldtrack/commands/${iot:Connection.Thing.ThingName}"
      ]
    },
    {
      "Effect": "Allow",
      "Action": "iot:Receive",
      "Resource": [
        "arn:aws:iot:eu-west-1:ACCOUNT_ID:topic/coldtrack/commands/${iot:Connection.Thing.ThingName}"
      ]
    }
  ]
}
```

Replace `ACCOUNT_ID` with your 12-digit AWS account ID.

2. Attach the policy and click **Create thing**.

### Step 4: Download Certificates

On the confirmation page, download all three files:

- **Device certificate** (`xxxxxxxxxx-certificate.pem.crt`)
- **Private key** (`xxxxxxxxxx-private.pem.key`)
- **Amazon Root CA 1** (or download separately from the link above)

**Save the private key immediately.** AWS will not let you download it again.

### Step 5: Find Your IoT Endpoint

1. In the AWS IoT Core console, go to **Settings** (bottom of the left sidebar).
2. Copy the **Device data endpoint**. It looks like:
   ```
   abcdefg12345-ats.iot.eu-west-1.amazonaws.com
   ```
3. Paste this value into `AWS_IOT_ENDPOINT` in `config.h`.

## How to Add Certificates to config.h

The ESP32 firmware reads certificates from `config.h` as `PROGMEM` C string constants. You need to paste the full PEM content (including the `-----BEGIN ...-----` and `-----END ...-----` lines) into the raw string literals in `config.h`.

### Example

If your `AmazonRootCA1.pem` contains:

```
-----BEGIN CERTIFICATE-----
MIIDQTCCAimgAwIBAgITBmyfz5m/jAo54vB4ikPmljZbyjANBgkqhkiG9w0BAQsF
ADA5MQswCQYDVQQGEwJVUzEPMA0GA1UEChMGQW1hem9uMRkwFwYDVQQDExBBbWF6
...many more lines...
rqXRfboQnoZsG4q5WTP468SQvvG5
-----END CERTIFICATE-----
```

Then in `config.h`, replace the placeholder:

```cpp
static const char AWS_ROOT_CA[] PROGMEM = R"EOF(
-----BEGIN CERTIFICATE-----
MIIDQTCCAimgAwIBAgITBmyfz5m/jAo54vB4ikPmljZbyjANBgkqhkiG9w0BAQsF
ADA5MQswCQYDVQQGEwJVUzEPMA0GA1UEChMGQW1hem9uMRkwFwYDVQQDExBBbWF6
...many more lines...
rqXRfboQnoZsG4q5WTP468SQvvG5
-----END CERTIFICATE-----
)EOF";
```

Repeat for `DEVICE_CERT` (your device certificate) and `DEVICE_PRIVATE_KEY` (your private key).

### Important Notes on Formatting

- Include the `-----BEGIN ...-----` and `-----END ...-----` lines exactly as they appear.
- Do **not** add extra blank lines or spaces inside the certificate block.
- The `R"EOF(...)EOF"` raw string literal syntax handles newlines automatically; you do not need to add `\n` escape sequences.

## Security Warnings

1. **Never commit certificates or private keys to Git.** Add the following to your `.gitignore`:

   ```gitignore
   # AWS IoT certificates
   esp32/certificates/*.pem
   esp32/certificates/*.pem.crt
   esp32/certificates/*.pem.key
   esp32/certificates/*.key
   esp32/certificates/*.crt

   # Config with embedded certs
   esp32/config.h
   ```

2. **Rotate certificates periodically.** If you suspect a key has been compromised, revoke the certificate immediately in the AWS IoT Core console and provision a new one.

3. **Use one certificate per device.** Do not share the same certificate and private key across multiple ESP32 units. Each device should have its own Thing and certificate pair.

4. **Store private keys securely.** In production, consider using the ESP32's secure boot and flash encryption features to protect the firmware image containing the keys.

5. **Restrict IoT policies.** Follow the principle of least privilege. The policy should only allow the specific MQTT topics that the device needs.

## Verifying Your Setup

After configuring certificates and flashing the firmware, open the Serial Monitor at 115200 baud. A successful connection looks like:

```
[WIFI] Connecting to "YourNetwork"...... connected!
[WIFI] IP:   192.168.1.42
[WIFI] RSSI: -55 dBm
[NTP]  Time synced: 2026-02-16 10:30:00 UTC
[MQTT] Configured for abcdefg12345-ats.iot.eu-west-1.amazonaws.com:8883
[MQTT] Connecting as "ESP32_TEST_002" (attempt 1/10)...
[MQTT] Connected to AWS IoT Core!
[MQTT] Subscribed: coldtrack/commands/ESP32_TEST_002
```

If you see `MQTT_CONNECT_FAILED` or TLS errors, verify:

- The endpoint URL is correct and includes the `-ats` suffix.
- All three certificates are properly pasted into `config.h`.
- The device certificate is activated in the AWS IoT Core console.
- The IoT policy is attached to the certificate.
- Your clock is synced (TLS requires accurate time).

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| `MQTT_CONNECTION_TIMEOUT` | Wrong endpoint or firewall blocking port 8883 | Verify endpoint; check network allows outbound 8883 |
| `MQTT_CONNECT_FAILED` | Certificate not activated or not attached to policy | Activate cert in console; attach IoT policy |
| `MQTT_CONNECT_UNAUTHORIZED` | IoT policy too restrictive | Update policy to allow required actions/resources |
| TLS handshake failure | Wrong Root CA or expired certificate | Re-download AmazonRootCA1.pem; check cert validity |
| WiFi connects but MQTT fails | NTP time not synced | Ensure NTP servers are reachable; check time output |
